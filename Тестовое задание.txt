Задание для тестирования.

Необходимо написать программу на С++ для ПК (серверную часть и/или клиентскую часть), которая выполняла бы следующие функции:

(серверная часть)
1) при включении ждать соединение связи по протоколу TCP/IP по указанному пользователем IP-адресу и TCP-порту.
2) ождать сообщений SCPI и отправляла соответствующие ответы. Список поддерживаемых сообщений описан в Приложении 1.
3) выводить в консоль тип поступившего запроса, сообщать также, если поступил неизвестный запрос.

(клиентская часть)
1) при включении соединяться с сервером по протоколу TCP/IP по указанному пользователем IP-адресу и TCP-порту.
2) отправлять на выбор пользователя сообщение SCPI серверу из списка приведённого в Приложении 1.
3) выводить в консоль ответ сервера, или сообщать, что ответа не поступило.


Приложение 1. Список сообщений SCPI.
1) запрос статуса устройства. 
запрос "SYSTem: STATe?"
ответ представляет собой значение bool (0 или 1)
2)запрос сведений по измерениям прибора.
запрос "MEASure: POINts?"
ответ представляет собой значение uint32 (значение 1000).
Значение в поле данных пакета размещается по принципу Big-Endian.
3) запрос значений измерений прибора.
запрос "MEASure: DATA?"
ответ представляет собой массив значений uint16 (количество 1000)
Значение в поле данных пакета размещается по принципу Big-Endian.

Сообщения SCPI должны поддерживать как короткий, так и длинный способ записи. Например, команда "SYSTem: STATe?" должна восприниматься в программе также, как и "SYST: STAT?".
